# What is HTML
![image](https://hackmd.io/_uploads/HJUOW0WI6.png)

```html
<form action="/submit_form" method="post">
  <!-- 表單中的各種輸入元素 -->
  <label for="username">用戶名稱：</label>
  <input type="text" id="username" name="username">

  <label for="password">密碼：</label>
  <input type="password" id="password" name="password">

  <input type="submit" value="提交">
</form>
```

上面的例子中，`<form>` 元素包含了兩個文本輸入框（用於輸入用戶名稱和密碼）以及一個提交按鈕。`action` 屬性指定了表單提交的目標 URL，而 `method` 屬性指定了提交表單時使用的 HTTP 方法（通常是 "post" 或 "get"）。

在表單中，各種輸入元素（如 `<input>`、`<select>`、`<textarea>` 等）用於收集不同類型的數據。表單還可以包含其他類型的元素，例如按鈕、標籤等。

表單的主要目的是讓用戶提交數據給網站，這樣網站就可以處理這些數據，執行相應的操作，比如用戶註冊、搜索、反饋等。表單的提交會觸發由 `action` 屬性指定的目標 URL 上的後端處理邏輯。

# Internal Css
這一部分的內容主要介紹了如何使用內部 CSS（internal CSS）來樣式化 HTML 元素，並強調其相對於內聯 CSS 的優勢。以下是分段和提取的重點：

1. **內部 CSS的基本使用：**
   - 在 `<head>` 標籤內，使用 `<style>` 開始標籤和結束標籤包裹 CSS 代碼。
   - 在 `<style>` 內，使用選擇器（selector）選擇要樣式化的 HTML 元素，然後定義相應的樣式屬性和值。

2. **範例程式碼：**
   - 以選擇器 `h1` 為例，設置樣式屬性 `color` 為 blue，`font-size` 為 3rem。
   - 以選擇器 `h2` 為例，設置樣式屬性 `color` 為 red，`font-size` 為 0.5rem。

3. **內部 CSS相對於內聯 CSS的優勢：**
   - 內部 CSS 的優勢在於不需要將樣式直接添加到 HTML 元素上，可以集中管理樣式。
   - 通過使用內部 CSS，對樣式的更改將立即應用於所有相應的 HTML 元素，使得管理和修改更加方便。

4. **程式碼的靈活性：**
   - 更改 CSS 的定義即可即時反映在所有相應的元素上，無需進行冗長的修改。
   - 內部 CSS 提供更清晰的結構，使得代碼更易於理解和維護。

總的來說，這部分內容強調了使用內部 CSS 的優點，並展示了它相對於內聯 CSS 的程式碼編寫和管理的優勢。

# External Css
![image](https://hackmd.io/_uploads/SyeqvsG8p.png)
![image](https://hackmd.io/_uploads/BJQqOizLp.png)

這一部分的內容主要介紹了使用外部 CSS 樣式表，以及相對於內聯 CSS 和內部 CSS 的優勢。以下是分段和提取的重點：

1. **建立外部 CSS 樣式表：**
   - 創建一個新的 CSS 檔案，通常以 `.css` 為擴展名，例如 `styles.css`。
   - 在 CSS 檔案中，使用選擇器來選擇 HTML 元素，然後定義相應的樣式屬性和值。

2. **使用 `<link>` 連結外部 CSS：**
   - 在 HTML 文件的 `<head>` 標籤中使用 `<link>` 元素，將 HTML 文件與外部 CSS 樣式表相連。
   - 在 `<link>` 元素中使用 `href` 屬性指定外部 CSS 檔案的路徑。

3. **相對路徑的重要性：**
   - 使用相對路徑指向外部 CSS 檔案，確保路徑的正確性。
   - 如果 CSS 檔案位於與 HTML 文件相同的目錄中，可以直接指定檔名；如果在子目錄中，則需要使用相對路徑。

4. **在多頁面中使用外部 CSS：**
   - 外部 CSS 的優勢在於可以在多個 HTML 頁面中共享相同的樣式，提高代碼的可維護性。
   - 修改外部 CSS 檔案中的樣式會立即反映在所有連結了該樣式表的 HTML 頁面上，減少了冗長的修改過程。

5. **建立 CSS 文件夾：**
   - 在較大的專案中，通常會將 CSS 檔案放入單獨的文件夾中，以保持代碼結構的清晰。
   - 使用 `<link>` 元素時，根據檔案的結構，指定相對路徑以連結到 CSS 文件夾中的樣式表。

總的來說，這一部分強調了外部 CSS 的使用方法，以及如何透過外部 CSS 提高代碼的組織性和可維護性。

這一部分的內容主要介紹了 CSS 的層級覆蓋（Override）概念，即在內聯、內部和外部 CSS 之間的權重優先順序。以下是分段和提取的重點：

1. **覆蓋概念介紹：**
   - 在 CSS 中，當相同的選擇器在不同的樣式表中出現時，可能會發生樣式的覆蓋。這種情況下，某些樣式會優先於其他樣式。
   - 有三種主要的樣式表：內聯 CSS（inline）、內部 CSS（internal）、外部 CSS（external）。

2. **外部 CSS 的強度：**
   - 外部 CSS 是最低優先順序的樣式表，即當相同的選擇器在內部或內聯樣式表中有樣式時，外部樣式表的樣式會被覆蓋。
   - 外部樣式表是全局性的，會影響連結的所有 HTML 頁面。

3. **內部 CSS 的覆蓋：**
   - 內部 CSS 的權重高於外部 CSS，當相同的選擇器在內部和外部樣式表中都有樣式時，內部樣式表的樣式會覆蓋外部的。
   - 內部樣式表僅影響包含它的 HTML 頁面。

4. **內聯 CSS 的最高權重：**
   - 內聯 CSS 具有最高的權重，即當相同的選擇器在內聯、內部和外部樣式表中都有樣式時，內聯樣式表的樣式會覆蓋其他兩者。
   - 內聯樣式是直接應用在 HTML 元素上的，僅影響該具體元素。

5. **樣式表的使用場景：**
   - 外部 CSS 適用於全局樣式，可在整個網站中共享相同的樣式，提高一致性。
   - 內部 CSS 適用於特定 HTML 頁面的樣式設定。
   - 內聯 CSS 適用於單個元素的具體樣式需求。

6. **權重優先順序：**
   - 內聯 CSS > 內部 CSS > 外部 CSS

總的來說，這一部分強調了在不同樣式表中樣式的優先順序，並指出了內聯 CSS 具有最高的權重，能夠覆蓋其他樣式。
# Basic Css Syntax
這一部分的內容主要介紹了 CSS 的基本語法，包括選擇器（selectors）、屬性（properties）、值（values）、聲明塊（declaration block）、CSS 規則（CSS rule）等相關概念。以下是分段和提取的重點：

1. **CSS 覆蓋原則再強調：**
   - 在前文提到過，內聯 CSS > 內部 CSS > 外部 CSS，內聯樣式最優先，外部樣式最低優先。

2. **CSS 規則的構成：**
   - 整個 CSS 規則由選擇器（selectors）、聲明塊（declaration block）組成。
   - 選擇器是用來選中 HTML 元素的部分，指定樣式作用的對象。
   - 聲明塊包含在大括號中，其中包含了一個或多個屬性及其相對應的值。

3. **CSS 規則的結構：**
   - CSS 規則整體被稱為 CSS rule。
   - 由選擇器和聲明塊構成，形成一個完整的樣式定義。

4. **CSS 屬性與值：**
   - 聲明塊中，每一個屬性都有相應的值，並由冒號分隔。
   - 屬性-值的組合被稱為聲明（declaration）。
   - 不同的屬性（property）控制不同的樣式特徵，如顏色（color）或字體大小（font size）等。

5. **CSS 註釋：**
   - 可以使用 `/* */` 來添加 CSS 註釋，用來解釋或分隔不同部分的代碼。
   - 註釋不會影響實際的樣式效果，僅為開發者提供說明。

6. **結構的嚴謹性：**
   - CSS 規則需要嚴格遵從結構，包括選擇器、大括號、聲明塊內的屬性-值對，以及分號作為屬性結束的標記。

7. **屬性與值的變化：**
   - 具體的選擇器、屬性和值會根據實際需求和設計要求而變化。
   - 認識基本語法是理解 CSS 的基石，後續的應用中會根據具體情境進行修改。

總的來說，這一部分深入介紹了 CSS 的基本語法和結構，強調了選擇器、屬性和值的角色，並提及了 CSS 規則的構成，為後續更複雜的 CSS 應用奠定了基礎。

# Rgba
![image](https://hackmd.io/_uploads/HJ81wGmL6.png)

`background` 和 `background-color` 是用於設定元素背景的 CSS 屬性，但它們有一些不同之處。

1. **`background` 屬性：**
   - `background` 屬性是一個綜合屬性，它允許你在一個聲明中同時設定多個與背景有關的屬性，如顏色、圖片、位置等。
   - 可以使用 `background` 屬性一次性指定多個值，例如：
     ```css
     background: #ff0000 url('background-image.jpg') center/cover no-repeat;
     ```
   - 上述例子中，設定了背景顏色、背景圖片、圖片位置、圖片大小調整以及不重複。

2. **`background-color` 屬性：**
   - `background-color` 屬性僅用於設定元素的背景顏色，而不包括其他與背景相關的屬性。
   - 這個屬性只控制背景的顏色，不涉及背景圖片、位置或其他屬性。
   - 例如：
     ```css
     background-color: #ff0000;
     ```
   - 上述例子中，僅設定了元素的背景顏色為紅色。

簡而言之，`background` 是一個用於設定元素完整背景的綜合屬性，而 `background-color` 僅用於設定元素的背景顏色。當你想要同時設定多個背景相關的屬性時，可以使用 `background` 屬性；如果只關心背景顏色，則使用 `background-color` 屬性即可。
這一部分主要介紹了在CSS中使用RGBA（帶透明度的RGB）顏色值的方法，包括說明RGBA的概念、語法以及如何運用它們。以下是分段和提取的重點：

`rgb` 和 `rgba` 都是 CSS 中用於表示顏色的屬性，但它們有一些區別。

1. **`rgb`：**
   - `rgb` 代表紅色（Red）、綠色（Green）、藍色（Blue）三個顏色通道。
   - 語法：`rgb(red, green, blue)`，其中 `red`、`green`、`blue` 的值範圍在 0 到 255 之間。
   - 例如，純紅色可以表示為 `rgb(255, 0, 0)`。

2. **`rgba`：**
   - `rgba` 也代表紅色、綠色、藍色三個通道，但多了一個額外的通道，即 alpha 通道，用於表示顏色的透明度。
   - 語法：`rgba(red, green, blue, alpha)`，其中 `red`、`green`、`blue` 的值範圍在 0 到 255 之間，而 `alpha` 的值範圍在 0（完全透明）到 1（完全不透明）之間。
   - 例如，半透明的紅色可以表示為 `rgba(255, 0, 0, 0.5)`，其中透明度為 0.5。

簡而言之，主要的區別在於 `rgba` 多了一個表示透明度的 alpha 通道，這使得你可以在顏色中添加透明效果。`rgba` 在處理背景、陰影或其他需要透明度控制的場景中很有用。而 `rgb` 則表示完全不透明的顏色。

   - 提到透明度通道的值範圍為0（完全透明）到1（完全不透明）。

# Calc Function
`min-height: calc(100vh - 100px)` 是一個 CSS 屬性的設定，用來指定元素的最小高度。這個屬性的具體效果取決於它應用的元素，但基本上它是這樣計算的：

- `100vh` 表示視窗高度的百分之百，即整個視窗的高度。
- `- 100px` 表示減去 100 像素。

所以，`min-height: calc(100vh - 100px)` 的含義是元素的最小高度應該為視窗高度的百分之百減去 100 像素。
<hr>

這樣的設定常見於需要確保元素至少佔滿整個視窗高度，但可以更高的情況。這在一些設計中是有用的，特別是在創建具有背景或內容的區域時。通常這樣的設定會確保即使內容很少，這個區域也不會縮得太小。

請注意，這樣的 CSS 規則可能需要根據你的實際情況進行微調，視你的設計需求而定。

# Padding

![image](https://hackmd.io/_uploads/Hy7VaA7Lp.png)

`padding` 是 CSS 中用來設定元素內邊距（即元素內部邊緣與內容之間的空間）的屬性。`padding` 可以應用到所有 HTML 元素。

屬性值可以是單一數值，也可以是四個值的空格分隔（上、右、下、左）。

### 單一數值的情況：

```css
padding: 10px;
```

這表示元素的上、右、下、左的內邊距都是 10 像素。

### 四個值的情況：


# Border

`border-style: solid;` 是 CSS 中用來設定元素邊框樣式的屬性。這個屬性指定了元素的四個邊的邊框樣式，並且 `solid` 表示使用實心的線條作為邊框。

使用這個屬性可以給元素添加一個單一的實心邊框。以下是一個例子：

```css
border-style: solid;
```

這會使元素的四個邊都具有相同的實心邊框。

如果你想要指定每個邊的邊框樣式，你可以使用 `border-top-style`、`border-right-style`、`border-bottom-style` 和 `border-left-style` 這四個單獨的屬性。例如：

```css
border-top-style: solid;
border-right-style: solid;
border-bottom-style: solid;
border-left-style: solid;
```

這樣的設定會讓每個邊都有單獨的實心邊框。

另外，你也可以同時使用 `border-width` 屬性來指定邊框的寬度，以及 `border-color` 屬性來指定邊框的顏色。這樣，你就可以更精細地控制元素的邊框外觀。例如

：

```css
border: 2px solid #000;
```

這表示元素的邊框寬度為 2 像素，並且邊框樣式為實心，邊框顏色為黑色（`#000`）。

`border` 是 CSS 中用來設定元素邊框的屬性，它可以用來指定邊框的寬度、樣式和顏色。通常，`border` 可以被拆分成三個子屬性：`border-width`、`border-style` 和 `border-color`，用來分別指定邊框的寬度、樣式和顏色。

### 使用 `border` 屬性的簡寫形式：

```css
border: 2px solid red;
```

上述代碼表示：

- `2px` 是邊框的寬度，表示邊框的寬度為 2 像素。
- `solid` 是邊框的樣式，表示邊框是實心的。
- `red` 是邊框的顏色，表示邊框的顏色為紅色。

### 拆分成子屬性：

```css
border-width: 2px;
border-style: solid;
border-color: red;
```

這三個子屬性可以獨立使用，也可以一同使用，視情況而定。如果只使用 `border` 屬性，則可以在一個屬性值中同時指定這三個子屬性的值，以簡化代碼。

`border` 屬性是設計網頁佈局和裝飾元素時常用到的屬性之一，它可以改變元素的外觀，使得網頁看起來更具吸引力。

# Display Property

`display: inline;` 是一個 CSS 屬性，用於指定元素的顯示方式為「行內元素」。行內元素不會獨占一行，而是僅占據其內容所需的寬度，不強制換行。

當你將 `display: inline;` 應用於一個元素時，這個元素會被視為行內元素，以下是一些行內元素的特點：

1. **不獨占一行：** 行內元素不會強制換行，相鄰的行內元素會在同一行上。

2. **僅占據內容寬度：** 行內元素的寬度僅占據其內容所需的寬度，不會強制指定寬度。

3. **不可以設定寬度和高度：** 通常，行內元素的寬度和高度由其內容確定，無法直接設定。

常見的行內元素包括 `<span>`、`<a>`、`<strong>`、`<em>` 等。

以下是一個簡單的例子：

```css
.my-inline-element {
  display: inline;
  color: #ff0000;
  text-decoration: underline;
}
```

這個 CSS 規則將 `.my-inline-element` 這個元素設定為行內元素，並指定了文字顏色和下劃線。

`display: block;` 是一個 CSS 屬性，用於指定元素的顯示方式為「區塊元素」。區塊元素會獨占一行，通常擁有自己的寬度和高度，不與其他元素共享水平空間。

當你將 `display: block;` 應用於一個元素時，這個元素會被視為區塊元素，以下是一些區塊元素的特點：

1. **獨占一行：** 區塊元素會開始一個新行，前後的元素都會排在不同的行上。

2. **擁有自己的寬度和高度：** 區塊元素通常擁有自己的寬度和高度，可以設定。

3. **可以設定寬度、高度、邊距等：** 你可以透過其他 CSS 屬性來調整區塊元素的寬度、高度、邊距等。

常見的區塊元素包括 `<div>`、`<p>`、`<h1>` 到 `<h6>` 等。

以下是一個簡單的例子：

```css
.my-block-element {
  display: block;
  width: 300px;
  height: 200px;
  background-color: #e0e0e0;
}
```

這個 CSS 規則將 `.my-block-element` 這個元素設定為區塊元素，並指定了寬度、高度和背景顏色。

`margin: 0 auto;` 是一種用於設置邊距的 CSS 寫法，通常應用在水平居中對齊元素的情境中。

這個屬性的意義是：
- `margin-top` 和 `margin-bottom` 被設置為 0（零）。

當將左右邊距都設置為 "auto" 時，這個元素會在水平方向上居中對齊，而上下邊距則設置為零，確保在垂直方向上不產生額外的空間。

```css
.container {
  width: 600px; /* 或其他寬度值 */

}
```

這樣 `.container` 元素就會在其容器中水平居中，且上下無額外邊距。

# Display:none, Opacity, Visibility
這是一個 CSS 的樣式定義，其中`.opacity-0`是一個類（class）名稱，代表這個樣式將應用於具有這個類的 HTML 元素。而`opacity: 0;`是這個樣式的屬性，它設置了元素的透明度為0。
簡單來說，當一個 HTML 元素擁有了`opacity-0`這個類，它的透明度將被設置為0，使得該元素變得完全透明，即不可見。這樣的樣式常被用於實現一些動畫效果或在某些情況下隱藏元素。

這是一個 CSS 的樣式定義，其中`.visibility`是一個類（class）名稱，代表這個樣式將應用於具有這個類的 HTML 元素。而`visibility: hidden;`是這個樣式的屬性，它將元素的可見性設置為隱藏。
簡單來說，當一個 HTML 元素擁有了`.visibility`這個類，它的可見性將被設置為隱藏，使得該元素在頁面上不可見，但仍將佔據其原本的空間。這樣的樣式通常用於在不移除元素的情況下隱藏它，或者用於在特定條件下調整元素的可見性。

<hr>
`visibility: hidden;` 是 CSS 中的屬性，用於控制元素的可見性。當設置為 `hidden` 時，元素仍然存在於文檔流中，但是它變得不可見。換句話說，元素佔據了頁面上的空間，但是它對用戶是不可見的。

示例：

```css
.hidden-element {
  visibility: hidden;
}
```

在這個例子中，帶有 `hidden-element` 類的元素將變得不可見，但仍將保留其佔據的空間。這與 `display: none;` 不同，後者將使元素不僅不可見，而且在頁面佈局中也不佔據空間。

需要注意的是，`visibility: hidden;` 只是隱藏元素，但它仍然會受到其他樣式和事件的影響。如果要完全從頁面中移除元素，包括其佔據的空間，可以使用 `display: none;`。

# Linear-Gradients
這是一個 CSS 樣式區塊，它應用於某個 HTML 元素，這個樣式區塊定義了該元素的外觀和布局。讓我們逐一解釋這些屬性的含義：

1. `height: 100vh;`
   - 設定元素的高度為視窗高度的 100%。`vh` 表示視窗高度的百分比單位。

2. `color: green;`
   - 設定元素的文字顏色為綠色。

3. `font-size: 60px;`
   - 設定元素的文字大小為 60 像素。

4. `display: flex;`
   - 將元素的顯示屬性設定為 Flexbox（彈性盒模型），這使得元素的子元素可以使用強大的布局功能。

5. `justify-content: center;`
   - 在 Flexbox 中，水平方向上對齊元素的內容，使其居中。

6. `align-items: center;`
   - 在 Flexbox 中，垂直方向上對齊元素的內容，使其居中。

7. `text-align: center;`
   - 設定元素內文本的水平對齊方式為居中。

這些樣式的組合通常用於創建一個在頁面上居中顯示並包含文本的區塊。例如，這可能是一個頁面的標題或者一個特別設計的區塊。
`background-attachment: fixed;` 是 CSS 中用來設定背景圖片滾動方式的屬性。

當設定 `background-attachment: fixed;` 時，背景圖片會固定在視窗的位置，不會隨著頁面的滾動而移動。這樣做的效果是，當你滾動網頁時，背景圖片會保持在屏幕上的相同位置，而不會像預設的滾動方式一樣隨著內容滾動。

例如：

```css
body {
  background-image: url('background.jpg');
  background-attachment: fixed;
}
```

這個例子中，`background-image` 設定了背景圖片的 URL，而 `background-attachment: fixed;` 則使得該背景圖片固定在視窗中，無論你滾動頁面內容，背景圖片都會保持在原地。

# Position Absolute
`position: relative;` 是 CSS 中用來指定元素定位方式的屬性之一。

當你對一個元素使用 `position: relative;` 時，它的位置會相對於它在正常文檔流（normal flow）中的位置進行調整。換句話說，元素仍然占據文檔流中的空間，但你可以使用額外的定位屬性（如 `top`、`right`、`bottom`、`left`）來微調它的實際位置。

```css
.element {
  position: relative;
  top: 10px;
  left: 20px;
}
```

上述代碼表示將 `.element` 元素向下移動 10 像素，向左移動 20 像素。這種相對定位的方式通常用於微調元素的位置，並與其他定位方式（如 `absolute`、`fixed`）搭配使用。

`position: absolute;` 是 CSS 中用來指定元素定位方式的屬性之一。

當你對一個元素使用 `position: absolute;` 時，它的位置會相對於其最接近的已定位（positioned）的祖先元素，如果沒有已定位的祖先元素，則相對於最初的包含塊（initial containing block，通常是瀏覽器的視窗）進行定位。

這種定位方式使得元素脫離正常文檔流，不再占據文檔流中的空間，可以自由漂浮在其他元素上方。你可以使用 `top`、`right`、`bottom`、`left` 屬性來指定元素相對於其包含塊的位置。

```css
.element {
  position: absolute;
  top: 50px;
  left: 100px;
}
```

上述代碼表示將 `.element` 元素相對於其包含塊（可能是某個祖先元素）的左上角位置向下移動 50 像素，向右移動 100 像素。

請注意，如果沒有已定位的祖先元素，`position: absolute;` 會相對於初始包含塊，這可能導致意外的結果，因此在使用時要格外小心。

# Transform:rotate()
在這一部分的課程中，教學者介紹了CSS的過渡（Transition）和動畫（Animation）屬性。以下是主要內容的分段整理：

**1. 過渡（Transition）屬性：**
   - 過渡用於在不同狀態之間平滑過渡元素的外觀變化。
   - 過渡屬性包括`transition-property`、`transition-duration`、`transition-timing-function`和`transition-delay`。
   - 可以簡單地將過渡應用於元素，使其在不同狀態之間產生動畫效果。
   - 教學者介紹了過渡的簡單實例，涉及顏色和位置的變化。

**2. 過渡時間函數（Transition Timing Function）：**
   - 過渡時間函數用於指定過渡的速度變化，控制動畫過程中的加速和減速。
   - 常見的過渡時間函數包括`ease`（默認值）、`linear`、`ease-in`、`ease-out`、`ease-in-out`等。
   - 這些函數影響過渡的速度曲線，例如“ease-in”表示開始時較慢，而“ease-out”表示結束時較慢。

**3. 動畫（Animation）屬性：**
   - 動畫提供更多選項，與過渡不同，動畫允許定義多個中間狀態。
   - 使用`@keyframes`定義動畫的中間狀態，並指定相應的百分比。
   - 動畫屬性包括`animation-name`、`animation-duration`、`animation-timing-function`、`animation-iteration-count`等。
   - 教學者展示了動畫的實例，包括背景顏色和位置的變化，以及如何使用`@keyframes`定義中間狀態。

**4. 過渡與動畫的區別：**
   - 過渡主要用於在兩個狀態之間實現平滑的變化，而動畫提供更多靈活性，允許定義多個中間狀態。
   - 過渡的屬性適用於單一元素的變化，而動畫則可應用於更複雜的場景，包括多個元素的同步動畫。

**5. 簡寫形式的動畫屬性：**
   - 動畫屬性可以使用簡寫形式，包括動畫名稱、持續時間、時間函數和迭代次數。
   - 使用簡寫形式時，可以更簡潔地定義動畫的基本屬性。

**6. 迭代次數與無限迭代：**
   - `animation-iteration-count`屬性用於指定動畫的迭代次數，可以是具體的次數或`infinite`表示無限迭代。
   - 使用無限迭代時，動畫將持續循環播放。

這些內容涉及CSS中實現動畫效果的基本概念，包括過渡和動畫的區別，過渡時間函數的應用，以及動畫的基本屬性和簡寫形式。

**1. 動畫填充模式介紹：**
   - 動畫填充模式是用來定義動畫結束後元素應該如何呈現的屬性。
   - 有三個主要的填充模式選項，分別是 `none`（預設值）、`forwards`（向前填充）和 `backwards`（向後填充）。

**2. 動畫的中間狀態設定：**
   - 使用 `@keyframes` 定義動畫的中間狀態，指定每個百分比處的元素樣式。
   - 以一個實例為例，這裡設定了在不同百分比處改變元素的不透明度（opacity）和水平位移（translateX）。

**3. 動畫的基本屬性設定：**
   - 使用 `animation` 屬性指定動畫的名稱、持續時間和迭代次數。
   - 在這個例子中，動畫名稱是 "move"，持續時間為 5 秒，迭代次數為 2 次。

**4. 向前填充（forwards）的應用：**
   - 向前填充表示當動畫結束時，元素將保留最後一個動畫狀態。
   - 在實例中，將動畫填充模式設為 `forwards`，使元素保留最後一個動畫狀態（opacity: 0.4; translateX: 20px）。

**5. 向後填充（backwards）的應用：**
   - 向後填充表示當元素未開始播放動畫時，即在動畫開始前，元素將使用第一個動畫狀態。
   - 在實例中，將動畫填充模式設為 `backwards`，使元素在動畫開始前就擁有第一個動畫狀態（opacity: 0）。

**6. 無填充（none）的應用：**
   - `none` 是動畫填充模式的預設值，表示不應用任何填充，元素將使用原始樣式。
   - 在實例中，使用 `none` 將使元素在動畫結束後返回原始樣式。

**7. 應用場景與適用性：**

1. **CSS 變數（Custom Properties）：**
   - CSS 變數允許在文件中存儲和重複使用值，提供了更靈活的樣式控制。
   - 變數的語法為 `--變數名: 值;`。
   - 變數可用於存儲顏色、過渡時間等。

2. **變數作用域：**
   - 全局作用域：變數在整個文檔中可用，可通過 `:root` 或 `html` 選擇器定義。
   - 局部作用域：變數在指定元素內部可用，通過元素選擇器定義。

3. **局部作用域示例：**
   - 在特定元素內部定義變數，例如 `<div>` 內的 `<p>` 元素。
   - 可以在內部元素中使用這些變數，並且它們僅在該範圍內生效。

4. **全局作用域示例：**
   - 在 `:root` 或 `html` 選擇器中定義全局變數，可供整個文檔使用。
   - 全局變數可被任何元素內部或外部使用。

5. **變數的重用和修改：**
   - 通過使用 `var(--變數名)` 語法，可以在文檔中的不同地方重用變數。
   - 修改變數的值會同時影響文檔中使用該變數的所有元素。

6. **CSS 變數與過渡效果：**
   - 變數也可用於定義過渡效果，例如定義過渡時間，這樣可以在整個項目中方便地進行調整。
   - 通過在 `transition` 屬性中使用變數，可以創建統一的過渡效果。

- 屬性繼承
- 局部變數
- 全局變數

# Browser Prefixes
![image](https://hackmd.io/_uploads/rJLIjJBLT.png)

**分段重點整理：**

1. **瀏覽器前綴（Browser Prefixes）** 是為了支援不同瀏覽器對最新CSS功能的實驗性實現而需要的。
2. 瀏覽器前綴通常使用在新的CSS3屬性上，以確保在各種瀏覽器中正確顯示。
3. **Can I Use（caniuse.com）** 是一個外部資源，用來查詢不同CSS屬性在各瀏覽器中的支援程度。
4. 在CSS屬性後添加瀏覽器前綴是一種保證跨瀏覽器兼容性的做法，但手動添加可能很繁瑣。
5. 使用工具如 **Autoprefixer** 或 **Online Autoprefixer** 可以自動生成並添加瀏覽器前綴。
6. 自動添加瀏覽器前綴可以透過構建工具（例如Gulp）或CSS壓縮工具（例如CleanCSS）實現。
7. 建議不手動添加瀏覽器前綴，以提高開發效率並保持清晰的CSS代碼。

**關鍵字與專業術語：**

1. **瀏覽器前綴（Browser Prefixes）**
2. **CSS3屬性**
3. **Can I Use**
4. **跨瀏覽器兼容性**
5. **Autoprefixer**
6. **Gulp**
7. **CleanCSS**
8. **自動添加前綴**

**分段重點整理：**

1. **語義化HTML（Semantic HTML）** 指的是使用特定的標籤結構內容，而不是僅僅使用`div`和`span`等通用標籤。
2. 語義標籤的語法與一般標籤相同，可以添加類別和ID，同樣可以透過CSS選擇器進行樣式設定。
3. 使用語義標籤有助於提供瀏覽器更清晰的內容結構，同時改善網站對屏幕閱讀器和鍵盤導航的可訪問性。
4. 建議不需過分擔心，因為隨著項目的進行，使用語義標籤會變得更加自然。
5. 在實際工作中，應該選擇恰當的語義標籤，例如`<header>`、`<section>`、`<footer>`等，來結構化網頁內容。
6. 使用語義標籤提高代碼可讀性，並讓瀏覽器和輔助技術更好地理解網頁內容。
